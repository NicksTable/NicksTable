Alpha Exilemet

Genre: First-Person Survival Exploration Simulator
Setting: Realistic Sci-Fi
Development Time: ~2 months, Ongoing Production Project
Team Size: 5 Developers

Team Composition:
2 Programmers • 2 Modelers • 1 UI Designer • 1 Animator • 1 VFX Artist • 1 Sound Designer • 1 Game Designer • 1 Producer

Role:
Gameplay Programmer • VFX Artist

Additional Contributions:
Model and Animation fixes, Testing.

Alpha Exilemet is a first-person survival and resource management game set on a hostile alien planet. Players explore an open world, gather materials, upgrade their equipment, and progressively repair their damaged spaceship, the Drakkar, in order to escape.

I was responsible for the core harvesting gameplay systems, including tools, resources, inventory interactions, and reactive visual feedback.

Systems Developed

  • Tool System
    • Base Tool architecture and derived tools
    • Pickaxe (solid resources)
    • Vacuum (liquid resources)
    • Rod (gas resources)
    • Tool stats and upgrades
    • Equip / unequip integration with Character and Animation Blueprint
    • Inventory integration
  • Reactive Tool Visuals
    • Dynamic effect color changes based on harvested resource colors
  • Resource System
    • Base Resource architecture and specialized resource classes
    • Solid, Liquid, and Gas resource types
    • Resource stats
    • Damage handling
    • Depletion and regeneration
    • Drop logic
  • Miscellaneous Gameplay Mechanics
    • Automatic movement acceleration when running toward the base hub

The project already includes a playable open world with multiple biomes, upgrade terminals, save/load systems, and a complete progression loop based on exploration, harvesting, and equipment upgrades.

Core Gameplay Loop

  • Explore five progressively more dangerous biomes
  • Harvest Solid, Liquid, and Gas resources
  • Sell materials for Krono (in-game currency)
  • Upgrade character and tool stats
  • Repair the Drakkar spaceship
  • Escape the planet

Key Features

  • Open-world exploration
  • Survival mechanics (oxygen management)
  • Upgradeable harvesting tools
  • Economy and progression terminals
  • Environmental challenge portals
  • Story-driven cutscenes

*As of the recording of the video many sounds such as harvesting sounds have yet to be added.

Implemented Tools

The harvesting system is built around a shared tool architecture. Each tool interacts with specific resource types while sharing common systems such as stats, inventory integration, upgrades, and equip/unequip logic.

  • Pickaxe — Harvests solid resources
  • Vacuum — Collects liquid resources
  • Rod — Extracts gaseous resources (animations WIP as of recording)
