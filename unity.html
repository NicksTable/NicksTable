Feline Fantasy

Genre: Tactical RPG
Inspiration: Final Fantasy Tactics
Development Time: ~2 months
Team Size: 6 Developers

Team Composition:
2 Programmers • 2 Artists (3D / 2D) • 1 VFX Artist • 1 Sound Designer • 1 Game Designer • 1 Producer

Role:
Gameplay Programmer • VFX Artist

Turn-based tactical RPG featuring cats as playable units. Each turn allows control of a single entity selected from a priority-based turn queue, creating a more dynamic flow than traditional “move all units” systems.

I was primarily responsible for the core gameplay architecture, including grid systems, pathfinding, entity logic, abilities, and combat infrastructure.

Systems Developed

  • Grid Manager & Grid Nodes
    • Procedural grid generation and removal with variable dimensions
    • Node height editing with snapping tools and custom gizmos
    • Node coloring based on layered status blending
  • Pathfinding
    • Dijkstra-based pathfinding with elevation-aware movement costs
    • Movement range limited by Entity.Speed
  • Status System
    • Generic mutable state containers with optional turn duration
    • Designed to avoid race conditions and support layered effects
  • Ability Framework
    • Abstract base class with customizable execution steps
    • Handles range visualization, selection, validation, and execution
  • Combat Manager
    • Centralized validator for ability requests
    • Hit processing and HitResult prediction
    • Supports reactive systems such as counters and interrupts
  • Entity System
    • Generic character class used for both units and destructible objects

In addition to programming, I created three early VFX to learn Unity's particle systems, primarily using VFX Graph.

Core Features

  • Turn-based tactical combat
  • Class triangle: Warrior → Archer → Mage
  • Two abilities per protagonist cat (standard + special)
  • Priority-based turn queue
  • Elevation affects movement cost
  • Tutorial level (Level 0)
  • Grid-based deployment phase

Planned Features

  • Base combat abilities
  • UI Manager
  • Tutorial Manager
  • Dialogue Manager
  • Animations, VFX, and sound integration

The current gameplay video showcases the tutorial level and early gameplay systems. Some cats visible in the deployment phase were spawned manually for testing purposes to demonstrate valid placement positions.

Early Movement Ability

Uses Dijkstra pathfinding constrained by Entity.Speed, displays reachable nodes, validates target selection, and executes movement. The final version is intended to interpolate unit movement through animations and node-to-node transitions.

Grid Tools

Editor tools for procedural grid generation, removal, node height editing, and visualization using custom gizmos.


Project Fear

Genre: Beat 'em up Survival Horror
Development Time: ~1.5 months
Team Size: 3 Developers (2 Programmers, 1 Game/Level/Sound Designer)

Role:
Programmer

Established the project's modular code architecture by separating responsibilities into reusable micro-components across all major gameplay systems, including:

  • Input
  • AI
  • Players
  • Characters (Entities)
  • Stats
  • Abilities (shared and fully compatible between both AI and Player characters)

Technical Notes

The gameplay shown in the video does not fully represent the capabilities of the implemented systems, as the project was still under active development at the time of recording. The visuals are also due for some heavy polishing and are not representative of the final quality we aimed to achieve, due to time constraints.


Systems Developed

  • Combat System
    • Centralized "arbiter" responsible for handling shared logic across multiple abilities
  • Status Effect System
    • AbilityStacks (prevents unintended ability stacking)
    • IFrames
    • HyperArmor
    • Stuns
    • Cooldowns
    • HitExceptions (used for counterattacks)
    • Custom temporary states for ability-specific behavior
  • Player System
    • Instantiates the Local Player and connects input components
  • Input Components
  • Ability Framework
    • LightAttack (Punch used by both the player and zombies)
    • Block (triggers a Parry if activated shortly before being hit by parryable attacks)
    • Walk
    • Dash
    • Jump
  • Stat Components
    • Health
    • Gravity
    • Friction
  • Collision Detection Components
    • Hitbox
    • Ground Check

Additional Contributions

  • General system integration
  • Architecture support
  • Minor gameplay and technical tasks
